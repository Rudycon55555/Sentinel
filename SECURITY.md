<h1>Sentinel Security Policy</h1><p></p><p>Sentinel is built with security as a core principle.<br>This document outlines how security issues should be reported, how they are handled, and the expectations for contributors and users.</p><p></p><h2>Reporting a Vulnerability</h2><p></p><p>If you discover a security vulnerability, please report it privately and responsibly.</p><p></p><p>Contact the maintainer at:</p><p></p><p><a target="_blank" rel="noopener noreferrer nofollow" href="mailto:rudrafordev@protonmail.com">rudrafordev@protonmail.com</a></p><p></p><p>Please include:</p><p></p><p>• A clear description of the issue<br>• Steps to reproduce<br>• Potential impact<br>• Any suggested fixes or mitigations</p><p></p><p>Do not open a public GitHub issue for security vulnerabilities.</p><p></p><h2>Supported Versions</h2><p></p><p>Security updates are provided for the latest stable version of Sentinel.<br>Older versions may not receive patches unless the issue is severe.</p><p></p><h2>Security Principles</h2><p></p><p>Sentinel follows these core principles:</p><p></p><p>• Defense‑in‑depth<br>• Minimal attack surface<br>• Strong cryptography defaults<br>• Secure-by-design APIs<br>• Strict input validation and sanitization<br>• Role‑based access control<br>• Session integrity and verification<br>• Safe file and path handling<br>• Transparent and responsible disclosure</p><p></p><h2>What to Expect After Reporting</h2><p></p><p>After submitting a report:</p><p></p><ol><li><p>You will receive an acknowledgment within 72 hours.</p><p></p></li><li><p>The issue will be investigated promptly.</p><p></p></li><li><p>A fix or mitigation plan will be developed.</p><p></p></li><li><p>You may be contacted for clarification or testing.</p><p></p></li><li><p>A patch will be released once the issue is resolved.</p><p></p></li><li><p>You will be credited unless you request anonymity.</p><p></p></li></ol><p></p><h2>Scope</h2><p></p><p>The following areas are considered part of Sentinel’s security scope:</p><p></p><p>• Authentication and session handling<br>• Cryptography (Argon2id, SHA‑256, AES‑256)<br>• Data handling (JSON, YAML, XML, SQL)<br>• Middleware and runtime enforcement<br>• Role‑based access control<br>• Tauri integration and JS bridging<br>• Template loading and injection<br>• Installer and project structure loader</p><p></p><h2>Out of Scope</h2><p></p><p>The following are not considered security vulnerabilities:</p><p></p><p>• Issues caused by developer misuse of Sentinel APIs<br>• Vulnerabilities in third‑party dependencies<br>• Tauri or OS‑level security issues<br>• Social engineering attacks<br>• Missing features or enhancements</p><p></p><h2>Responsible Disclosure</h2><p></p><p>Sentinel follows a responsible disclosure model.<br>Please do not publicly share details of a vulnerability until a fix has been released.</p><hr><p></p>
